1. What are the differences between map filter find forEach in javascript?

Answer:
  • 1. map Returns a new array with the same length as the original array, where each element is transformed according to a function you provide.
  • 2. filter Returns a new array with all elements that pass a test implemented by a function you provide.
  • 3. find Returns the first element in the array that passes a test implemented by a function you provide, or undefined if no such element is found.
  • 4. forEach Calls a function for each element in the array, but does not return anything.

2. What are the differences between localStorage sessionStorage cookie in Browser?

Answer:
  • 1. localStorage Stores key/value pairs in the browser's storage, and persists even after the user closes the browser window or navigates away from the site. The data is associated with a specific domain and is not shared across different domains. localStorage can store more data than cookies, typically up to 5-10 MB.
  • 2. sessionStorage Similar to localStorage, but the data is only accessible for the duration of the browser session (i.e. until the user closes the browser window or tab). The data is also associated with a specific domain and is not shared across different domains. sessionStorage can store more data than cookies, typically up to 5-10 MB.
  • 3. Cookie Stores small amounts of data in the browser's storage, typically limited to 4 KB per cookie. Cookies can be set with an expiration time and can be accessed by both the server and the client, making them useful for storing user-specific data or tracking user behavior across multiple sessions or sites.

3. What is Event loop in javascript and How its works?

Answer: The event loop is a key concept in JavaScript that allows the language to handle asynchronous events in a non-blocking way. It is essentially a loop that continuously checks the JavaScript runtime's call stack and task queue, and executes tasks in the task queue when the call stack is empty.

How it Work:
  • 1. The event loop continuously checks the call stack to see if there are any functions that need to be executed.
  • 2. If the call stack is empty, the event loop looks for any pending events or callbacks in the event queue.
  • 3. If there are any events or callbacks in the queue, the event loop takes the first one and pushes it onto the call stack.
  • 4. The function associated with the event or callback is then executed.
  • 5. If the function is asynchronous, it may register a new event or callback with the event loop and return immediately.
  • 6. The event loop then repeats this process, checking the call stack for any pending functions and processing any events or callbacks in the queue.

1. What is the purpose of quizTimer() function?

Answer:

The purpose of the quizTimer function is to display a timer that counts up from 0 seconds and updates every second. It does this by using the setInterval method to call an anonymous function that updates the timer HTML element with the current time every second.

The function takes one parameter dismiss, which is a boolean value. If dismiss is true, it clears the timer using the clearInterval method and stops the timer from running. This is useful when the timer needs to be stopped manually, for example when the quiz is over or the user has clicked a "Submit" button.

The function also adds a visual effect to the timer by changing the color of the text every 60 seconds. If the seconds count is a multiple of 60, it toggles the color of the text between two shades of red by adding or removing CSS classes from the HTML element. This is a nice touch to add some visual interest to the timer and keep the user engaged.

Answer:

The purpose of the chooseQuiz function is to handle the user's selection of a quiz answer and store the selected answer in an answers array. If an answer object already exists, it updates the givenAns property of that object with the user's new answer using the splice method to replace the existing object with a new object that has the updated givenAns property. If an answer object does not exist, it creates a new object with the id, question, and givenAns properties and pushes it into the answers array.
